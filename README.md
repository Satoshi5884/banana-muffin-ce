# banana-muffin-chrome

`banana-muffin-chrome` は、Chrome のサイドパネルで動作する Manifest V3 ベースの画像生成拡張です。  
Gemini API と fal.ai API を切り替えて使え、単発の画像生成、IMG連、ギャラリー管理、ノート管理を 1 つの UI にまとめています。

この配布フォルダはビルド済みです。開発ビルドや追加の `npm install` は不要で、そのまま Chrome に読み込めます。

## 主な機能

- 画像生成
  - Gemini / fal.ai を切り替えて画像生成
  - `nano-banana` / `nano-banana 2` / `nano-banana-pro` を利用可能
  - アスペクト比、解像度、候補数、ネガティブプロンプトを設定可能
  - 画像入力付きの I2I に対応
- IMG連
  - `IMG01` 形式のプロンプトブロックを順番に処理
  - 共通参照画像またはブロックごとの個別参照画像を指定可能
  - 生成結果をまとめて ZIP 出力、またはセットへ自動保存
- ギャラリー
  - 生成画像や手動取り込み画像をローカル保存
  - セット分け、お気に入り、モデル別フィルタ、ZIP 出力に対応
- ノート
  - Markdown ベースのノートをローカル保存
  - プロンプトをノートに保存、ノート内容を生成欄へ読み込み可能
- 設定
  - Gemini API キー / fal.ai API キー保存
  - デフォルトプロバイダー、モデル、候補数、自動保存の設定
  - 利用履歴とストレージ使用量の確認

## 導入方法

1. このフォルダを手元に展開します。
2. Chrome で `chrome://extensions/` を開きます。
3. 右上の「デベロッパー モード」を有効にします。
4. 「パッケージ化されていない拡張機能を読み込む」を押します。
5. この `banana-muffin-ce` フォルダを選択します。
6. 拡張機能アイコンをクリックすると、サイドパネルで `banana-muffin-chrome` が開きます。

## 初期設定

1. 拡張機能を開き、`設定` タブを開きます。
2. `Gemini APIキー` を入力して保存します。
3. fal.ai を使う場合は `fal.ai APIキー` も入力して保存します。
4. 必要に応じて以下を設定します。
   - デフォルトプロバイダー
   - デフォルトモデル
   - デフォルト候補数
   - 自動保存

補足:

- Gemini API の確認ボタンがあります。
- `自動保存` を有効にすると、生成完了時にギャラリーへ自動保存されます。

## 画像生成機能

画像生成タブは、この拡張の中心機能です。

- 生成スロット
  - 最大 20 スロットまで追加可能
  - 新しいスロット作成時には、現在の入力状態を引き継げます
- 対応プロバイダー
  - Gemini
  - fal.ai
- 対応モデル
  - `nano-banana`
  - `nano-banana-2`
  - `nano-banana-pro`
- 指定できる項目
  - プロンプト
  - ネガティブプロンプト
  - アスペクト比
  - 解像度
  - 候補数 `1〜4`
  - fal.ai 利用時の出力形式 `PNG / JPEG / WebP`
- I2I 入力
  - 最大 14 枚まで画像入力可能
  - ローカルファイル追加
  - ギャラリーから選択
  - 画像セットから一括取り込み
- ノート連携
  - 現在のプロンプトをノート化
  - 既存ノートを読み込んで再利用
- 保存
  - 自動保存オフ時でも、生成結果ごとに手動保存可能
  - 保存先はギャラリーです

Gemini 側では内部的に以下の画像モデルを利用しています。

- `nano-banana`: `gemini-2.5-flash-image`
- `nano-banana-2`: `gemini-3.1-flash-image-preview`
- `nano-banana-pro`: `gemini-3-pro-image-preview`

fal.ai 側では以下のエンドポイントを利用しています。

- `fal-ai/nano-banana`
- `fal-ai/nano-banana-2`
- `fal-ai/nano-banana-pro`

## IMG連機能

`IMG連` タブは、複数のプロンプトブロックを順番に処理するための機能です。  
同じ元画像から複数パターンを連続生成したい場合に向いています。

### 基本の使い方

1. `IMG連` タブへ移動します。
2. テキスト欄に IMG ブロックを書き込むか、`.txt` / `.md` ファイルを読み込みます。
3. モデル、プロバイダー、アスペクト比、解像度などを設定します。
4. 参照画像を設定します。
   - 共通画像を 1 枚指定
   - もしくは各 IMG ブロックごとに個別指定
5. 実行すると、ブロック順に 1 枚ずつ生成されます。

### ブロック形式

以下のような fenced code block 形式で記述します。

````text
```IMG01
柔らかい朝の光が差し込む、木目調の机の上に置かれたマグカップ
```

```IMG02
同じ構図のまま、雨上がりの青いトーンに変えてください
```
````

ポイント:

- `IMG01`, `IMG02`, `IMG03` のようなラベルを使います
- 本文が空のブロックは警告対象です
- 重複ラベルも警告対象です
- `VID` ブロックはこの拡張ではスキップされます
- 最初の画像ブロックが `IMG01` でない場合も警告が出ます

### IMG連でできること

- サンプルテンプレートを即時読込
- `.txt` / `.md` ファイルからプロンプト読込
- 実行中の進捗表示
- 実行途中のキャンセル
- 結果画像の個別ダウンロード
- 結果画像の個別保存
- 完了画像の ZIP 一括ダウンロード
- セット自動保存

補足:

- ファイルから読み込んだ場合、セット自動保存時のセット名にはファイル名が優先して使われます
- 手入力の場合は `IMG連 YYYY-MM-DD HH:mm` 形式のセット名になります

## ノート機能

`ノート` タブは、プロンプトやメモを Markdown で整理するためのローカルノート機能です。

- 新規ノート作成
- ノート一覧表示
- 自動保存
  - 編集後およそ 500ms で保存
- ノート削除
- タイトル自動生成
  - 先頭行や見出しをもとに最大 60 文字で自動決定
- 生成タブとの連携
  - 現在のプロンプトをノートとして保存
  - ノート内容を生成欄へ読み込み

このため、プロンプト管理やバリエーション管理をローカルだけで回しやすい構成になっています。

## ギャラリー機能

ギャラリーには生成結果や手動で取り込んだ画像が保存されます。

- モデル別フィルタ
- セット別フィルタ
- お気に入り切り替え
- 手動画像取り込み
- セット作成 / セット削除
- セットへの割り当て
- 単体ダウンロード
- セット単位の ZIP ダウンロード

IMG連や画像生成の保存先として使う前提の機能なので、運用上かなり重要です。

## データ保存先

この拡張は、基本的にローカル保存中心です。

- `chrome.storage.local`
  - Gemini API キー
  - fal.ai API キー
  - 設定
  - 最後に開いていたタブ
- IndexedDB `banana-muffin-ext`
  - 画像
  - 画像セット
  - ノート
  - 利用履歴

つまり、拡張版はブラウザローカルで完結する運用が前提です。  
ブラウザプロファイルを消した場合は、これらのローカルデータも失われる可能性があります。

## 権限と通信先

`manifest.json` で使われている権限は最小限です。

- `storage`
- `sidePanel`

ホスト権限:

- `https://generativelanguage.googleapis.com/*`
- `https://fal.run/*`
- `https://*.fal.ai/*`

つまり、この拡張は任意の Web サイトへ広くアクセスするタイプではなく、主に画像生成 API とローカル保存を扱う構成です。

## web版の紹介

公式 web 版: [https://banana-muffin.com/](https://banana-muffin.com/)

Chrome 拡張版が「ローカルで素早く画像生成・管理する」用途に寄っているのに対し、web 版は機能範囲が広く、画像管理に加えて動画生成ワークフローにも対応しています。

公開されている web 版フロントエンド上では、少なくとも以下の系統の機能が確認できます。

- 画像一覧
- 画像セット管理
- 動画一覧
- 動画フレーム抽出
- クラウド保存
- `Seedance` などを含む動画系モデルの利用導線

そのため、静止画中心の作業を拡張版で回し、動画生成やクラウド管理を web 版で扱う、という使い分けがしやすい構成です。
